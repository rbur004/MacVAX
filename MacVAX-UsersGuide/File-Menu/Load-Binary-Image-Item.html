Load Binary Image Menu Item

    The Load Binary Image Item displays a Macintosh Standard dialog box prompting the user for a file to boot. Files of ALL types will be displayed for selection.

    All running processes are terminated and the processor is initialized to a know state. The file is loaded into VAX memory at Physical address 0x00000000. No symbol table information is loaded. The Stack Pointer is set to Physical address 0x0000F000. The processor is running in kernel mode on the Interrupt Stack with virtual memory turned off. The System Control Block Base Register will contain the physical address 0x00000000. The PC will be set to 0x00000000.

    The boot process is not started until the user selects "Go" from the "Run"" menu.

    This differs from "Load Boot Image" in that the file is assumed not to have been produced by the assembler AS. Its primary function is to load code images from real VAX processors.
